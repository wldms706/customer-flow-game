# PRD.md  
## 프로젝트명  
Customer Flow Game (무료라이브 전용 미니 게임)

---

## 1. 목적 (Purpose)

뷰티샵 원장 대상 무료라이브에서  
"손님은 랜덤으로 오지 않는다"는 메시지를  
**설명 ❌ / 체험 ⭕**으로 전달하기 위한  
인터랙티브 미니 게임 제작.

본 게임은  
강의형 설명 없이도  
👉 고객 유입 구조의 부재를 스스로 인식하게 만들고  
👉 이후 330만 원 실행 시스템 프로그램으로  
자연스럽게 전환시키는 것을 목표로 한다.

---

## 2. 타겟 사용자 (Target)

- 반영구 / 에스테틱 / 뷰티샵 원장
- 기술은 있으나
  - 고객 유입 경로를 명확히 설명 못함
  - 광고/콘텐츠/리타겟 구조 이해 부족
- 무료라이브 시청 중 참여자

---

## 3. 핵심 메시지 (Core Insight)

- 고객은 "우연히" 오는 것이 아니라  
  **항상 정해진 경로를 통해 유입된다**
- 대부분의 원장은  
  이 경로를 인식하지 못하고 있으며  
  **통제 가능한 지점도 모른다**
- 구조를 가진 매장만  
  같은 돈으로 반복 유입을 만든다

---

## 4. 게임 컨셉 (Game Concept)

### 타이틀
"오늘 이 손님, 어디서 왔을까요?"

### 플레이 방식
- 질문 제시
- 선택지 클릭
- 선택에 따라 고객 유입 경로 시각화
- 통제 가능 / 불가능 지점 명확화

---

## 4-1. 정숙이 페르소나 (Customer Persona: 정숙이)

### 기본 설정
- **이름**: 김정숙 (38세)
- **직업**: 직장인 (사무직)
- **상황**: 최근 피부 고민이 생겨 반영구/에스테틱 시술을 고려 중
- **특징**:
  - 신중한 성격, 후기를 꼼꼼히 확인
  - 가격보다 신뢰를 중시
  - 결정까지 평균 2주 소요

---

## 4-2. 정숙이의 신뢰 여정 (Trust Journey)

정숙이가 "낯선 매장"에서 "예약 고객"이 되기까지의 심리 변화와 행동을 단계별로 시각화합니다.

### 🔴 STAGE 1: 인지 (Awareness)
| 항목 | 내용 |
|------|------|
| **정숙이 상태** | "어디서 하지?" 막연한 탐색 |
| **심리** | 😐 무관심 / 정보 수집 모드 |
| **행동** | 인스타 피드 스크롤, 네이버 검색 |
| **신뢰도** | ⬜⬜⬜⬜⬜ (0%) |
| **이탈 확률** | 90% - 대부분 그냥 지나침 |
| **원장님 개입점** | ❌ 통제 불가 (알고리즘 의존) |

**정숙이 내면의 목소리:**
> "반영구 눈썹 어디서 하지... 일단 검색이나 해볼까"

---

### 🟠 STAGE 2: 관심 (Interest)
| 항목 | 내용 |
|------|------|
| **정숙이 상태** | "이 샵 괜찮아 보이네?" 첫 관심 |
| **심리** | 🤔 호기심 / 탐색적 관심 |
| **행동** | 프로필 클릭, 게시물 2-3개 확인 |
| **신뢰도** | ⬜⬜⬜⬜⬜ → 🟨⬜⬜⬜⬜ (20%) |
| **이탈 확률** | 70% - 콘텐츠가 별로면 이탈 |
| **원장님 개입점** | ⚠️ 부분 통제 (콘텐츠 퀄리티) |

**정숙이 내면의 목소리:**
> "사진은 괜찮은데... 진짜 실력 있는 곳인지 모르겠네"

**이 단계에서 정숙이가 확인하는 것:**
- 시술 전후 사진 퀄리티
- 게시물 업로드 빈도
- 전체적인 분위기

---

### 🟡 STAGE 3: 탐색 (Consideration)
| 항목 | 내용 |
|------|------|
| **정숙이 상태** | "더 알아봐야겠다" 적극적 정보 수집 |
| **심리** | 🧐 비교 분석 / 검증 모드 |
| **행동** | 후기 검색, 다른 샵과 비교, 가격 확인 |
| **신뢰도** | 🟨⬜⬜⬜⬜ → 🟨🟨⬜⬜⬜ (40%) |
| **이탈 확률** | 50% - 후기/정보 부족 시 이탈 |
| **원장님 개입점** | ✅ 통제 가능 (홈페이지, 후기 관리) |

**정숙이 내면의 목소리:**
> "후기가 좀 있긴 한데... 다른 데랑 비교해봐야지"
> "가격은 어디서 확인하지? 카톡으로 물어봐야 하나..."

**이 단계에서 정숙이가 확인하는 것:**
- 네이버 플레이스 후기
- 블로그 체험 후기
- 가격 정보 접근성
- 원장 경력/자격

**⚠️ 이탈 트리거:**
- 가격 정보가 없음 → "귀찮아, 다른 데 볼래"
- 후기가 너무 적음 → "검증이 안 됐네"
- 답변이 느림 → "바쁜가? 관리 잘 해줄까?"

---

### 🟢 STAGE 4: 의향 (Intent)
| 항목 | 내용 |
|------|------|
| **정숙이 상태** | "여기로 거의 마음 굳혔어" |
| **심리** | 😊 긍정적 기대 / 확신 필요 |
| **행동** | 카카오톡 문의, 상담 예약 고민 |
| **신뢰도** | 🟨🟨⬜⬜⬜ → 🟩🟩🟩⬜⬜ (60%) |
| **이탈 확률** | 30% - 상담 응대에서 실망 시 이탈 |
| **원장님 개입점** | ✅ 통제 가능 (상담 퀄리티, 응대 속도) |

**정숙이 내면의 목소리:**
> "일단 문의해볼까... 답변 보고 결정해야지"
> "상담 예약하면 부담스러우려나?"

**이 단계의 핵심:**
- 첫 응대의 온도
- 답변 속도 (30분 이내 vs 하루 뒤)
- 강매 느낌 여부

---

### 🔵 STAGE 5: 결정 (Decision)
| 항목 | 내용 |
|------|------|
| **정숙이 상태** | "예약할게요!" |
| **심리** | 😄 확신 / 기대감 |
| **행동** | 예약 확정, 날짜 조율 |
| **신뢰도** | 🟩🟩🟩⬜⬜ → 🟩🟩🟩🟩⬜ (80%) |
| **이탈 확률** | 10% - 노쇼 가능성만 존재 |
| **원장님 개입점** | ✅ 통제 가능 (예약 확정 프로세스, 리마인드) |

**정숙이 내면의 목소리:**
> "다음 주 토요일로 예약해야겠다"
> "떨리긴 하는데 기대된다!"

---

### 💜 STAGE 6: 충성 (Loyalty) - 시스템이 있는 매장만
| 항목 | 내용 |
|------|------|
| **정숙이 상태** | "다음에도 여기서!" + 지인 추천 |
| **심리** | 🥰 만족 / 신뢰 / 애착 |
| **행동** | 재방문, 후기 작성, 지인 소개 |
| **신뢰도** | 🟩🟩🟩🟩⬜ → 🟩🟩🟩🟩🟩 (100%) |
| **이탈 확률** | 5% - 거의 고정 고객화 |
| **원장님 개입점** | ✅ 통제 가능 (리타겟, CRM, 후기 요청) |

**정숙이 내면의 목소리:**
> "친구가 어디서 하냐고 물어보면 여기 추천해줘야지"
> "다음 리터치도 여기서 해야겠다"

---

## 4-3. 신뢰 이탈 포인트 (Trust Breakdown Points)

각 단계에서 정숙이가 이탈하는 구체적인 상황:

| 단계 | 이탈 원인 | 정숙이의 반응 |
|------|----------|--------------|
| 인지→관심 | 콘텐츠가 눈에 안 띔 | "그냥 스크롤..." |
| 관심→탐색 | 프로필이 허술함 | "뭔가 아마추어 같아" |
| 탐색→의향 | 정보 부족, 후기 없음 | "검증이 안 됐네, 패스" |
| 의향→결정 | 느린 응대, 불친절 | "다른 데 알아봐야겠다" |
| 결정→충성 | 사후 관리 없음 | "한 번 가고 끝이네" |

---

## 4-4. 시스템 유무에 따른 정숙이의 여정 비교

### ❌ 시스템 없는 매장
```
인지 → 관심 → [이탈]
         ↓
      다른 샵으로 이동
```
- 정숙이를 다시 만날 방법이 없음
- 한 번 놓치면 영영 끝

### ✅ 시스템 있는 매장
```
인지 → 관심 → 이탈
               ↓
         [픽셀이 기록]
               ↓
         [리타겟 광고]
               ↓
         다시 관심 → 탐색 → 의향 → 결정 → 충성
```
- 정숙이가 이탈해도 다시 만남
- 평균 7회 노출 후 전환

## 5. 화면 구성 (Screen Flow)

### STEP 1. 인트로 화면

- 텍스트:
  "오늘 방문한 고객 1명"
  "이 고객은 어디서 왔을까요?"

- 버튼 (5개):
  1. 네이버 플레이스
  2. 인스타그램
  3. 지인 소개
  4. 카카오톡 문의
  5. 기억 안 남

- UX:
  - 버튼 클릭 시 다음 화면 전환
  - 클릭 애니메이션 필수

---

### STEP 2. 고객 유입 경로 시각화

- 선택한 경로에 따라
  고객 이동 플로우를 선(Line)으로 연결하여 표시

예시:
인스타그램 선택 시

인스타 콘텐츠
   ↓
프로필 클릭
   ↓
홈페이지 / 카카오톡
   ↓
문의
   ↓
예약

- 애니메이션:
  - 선이 순차적으로 그려지는 방식
  - 각 단계 텍스트 Fade-in

---

### STEP 3. 통제 영역 강조

- 텍스트 질문:
  "이 중에서 원장님이 통제할 수 있는 건 몇 개일까요?"

- 각 단계 위에 아이콘 표시:
  ❌ 통제 불가 (알고리즘 / 플랫폼)
  ⭕ 통제 가능 (홈페이지 / 픽셀 / 리타겟)

- 강조 효과:
  통제 불가 영역 회색 처리
  통제 가능 영역 컬러 강조

---

### STEP 4. 시스템 유무 비교 화면

#### 시스템 ❌ 매장
콘텐츠 → 문의 → 끝

#### 시스템 ⭕ 매장
콘텐츠 → 픽셀
        ↓
      광고
        ↓
     리타겟
        ↓
      문의
        ↓
     재방문

- 좌/우 비교 구조
- 차이점 명확히 드러나는 컬러 대비

---

### STEP 5. 인식 전환 문장 노출

- 고정 문장:
  "문제는 손님이 없는 게 아니라  
   손님이 남지 않는 구조입니다."

- CTA 없음 (라이브용이므로)

## 6. 기능 요구사항 (Functional Requirements)

- 버튼 클릭 이벤트

## 7. 비기능 요구사항 (Non-Functional)

- 외부 로그인/저장 기능 없음
- 광고/픽셀 삽입은 후속 버전

## 11. 핵심 원칙 (Principles)

- 설명보다 체험
- 설득보다 인식
- 정보보다 구조
- 감정보다 시스템

## 12. 한 줄 요약

이 프로젝트는  
**"왜 매출이 불안한지"를 알려주는 게임이 아니라  
"왜 구조가 필요한지"를 깨닫게 만드는 장치다.**

### Additional Insight

고객 유입은 단일 행동이 아니라
여러 단계의 누적 확률 구조이다.
각 단계 중 하나라도 이탈이 발생하면
전체 전환 확률은 급격히 감소한다.

본 게임은
"여기서 끝난다"가 아니라
"여기서 확률이 줄어든다"는 사실을
체험시키는 것을 목표로 한다.
